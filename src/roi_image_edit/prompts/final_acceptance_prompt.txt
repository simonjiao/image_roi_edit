请对最终局部文字修图结果进行验收。

输入包括：
1. 紧凑验收图，包含原 ROI、最终 ROI、同排参考文字和背景补丁区；
2. 最终使用的参数；
3. `vision_brief` 短摘要。

最终使用参数：
{final_params}

vision_brief 短摘要：
{hard_check_report}

`vision_brief` 中包含当前候选的 `stage_context`、本地确定性检查摘要和历史人工拒绝目标。完整本地报告只用于本地审计，不作为视觉模型上下文全文输入。如果 `stage_context.blocking_stage`
不为空，视觉验收必须围绕该阶段判断失败原因；参数建议只能使用
`stage_context.allowed_patch_keys` 中允许的参数族，不能把
`stage_context.blocked_patch_keys` 作为当前轮主修复方向。
`local_deterministic_checks` 中的 pass 字段只表示对应本地门禁状态，不等于最终可交付结论；
即使 `strict_gate_pass=true` 且 `stage_gate_pass=true`，仍必须独立按图像判断文字是否自然。
如果历史人工拒绝目标激活，不能因为本地 pass 字段或没有 `blocking_stage` 就推断可交付。
`stage_context.profile_constraints` 是当前 profile 的约束。如果
`vision_context_scale=magnified`，必须按放大对比优先验收字体结构、笔画体量和
低分辨率边缘；如果 `enable_photo_texture=false` 或 `enable_photo_warp=false`，
不要把照片质感或 photo_warp 作为当前主修复方向。
公开阶段只能是：`hard_boundary`、`text_shape`、`ink_gray_balance`、`photo_texture`、`background_cleanup`。

硬性验收规则：
1. 如果输出尺寸与原图尺寸不一致，必须 pass=false。
2. 如果 ROI 外变化像素数不为 0，必须 pass=false。
3. 如果边缘变化像素数不为 0，必须 pass=false。
4. 如果 field_label_text、field_separator_text、protected_texts、protected_boxes 或任何非目标文字被修改，必须 pass=false。
5. 如果图片被拉伸、缩放、裁剪、重采样，必须 pass=false。
6. 只有硬校验全部通过，才进入视觉验收。

视觉验收重点：
1. target_text 是否自然替代 source_text；
2. 每个目标字符是否贴近对应旧槽位或合理扩展槽位；
3. 新文字的位置、字距、基线是否接近原图；
4. 字体风格是否与原图一致或至少不突兀；
5. 字号是否明显偏大或偏小；
6. 笔画粗细是否明显不一致；
7. 黑度是否明显过黑或过浅；
8. 清晰度是否明显过锐或过糊；
9. 背景是否有明显补丁、残影、涂抹感或接缝；
10. ROI 边界是否有突兀过渡；
11. 整体是否可以接受为同一张图中的文字。

验收口径：
- pass：硬校验通过，视觉上无明显突兀，可交付。
- marginal：硬校验通过，但视觉上有轻微问题；可交付但建议小幅优化。
- fail：硬校验失败，或视觉上明显不自然。

必须按阶段给出失败原因：
1. `text_shape`：字体结构、字号、字槽、字距、基线、局部倾斜、笔画身体；
2. `ink_gray_balance`：真黑核心、中灰笔画、外灰边；
3. `photo_texture`：照片/扫描模糊、锯齿断裂、噪声、压缩质感；
4. `background_cleanup`：旧字残留、发白、过平滑、涂抹、接缝。

如果 `text_shape` 存在 hard-blocking issues，不能用“降黑、加模糊、加噪声、压缩”作为主要修复方向。
如果 `stage_status.text_shape.pass=true` 或 `pass_with_deferred=true`，且存在 `deferred_issues` / `deferred_to_stage`，这些问题不能作为最终通过理由，但当前修复方向应跟随 `deferred_to_stage` 和本地 `blocking_stage`，同时保证字体、字号、字槽、基线、字距和严重姿态指标不回退。
如果 `ink_gray_balance` 未通过，必须明确是 `too_dark`、`too_bold`、`too_light` 还是 `too_thin`，不要混用。
禁止把“核心过黑、核心黑度偏重、黑芯过量、too_dark”写成“核心不够黑”。
如果本地阶段上下文禁止某参数，你仍可在 reason 中描述该现象，但不要在
`suggested_patch` 或 `parameter_suggestions` 中建议该参数。
如果 `vision_brief.historical_false_pass_target.active=true`，必须逐个判断其中的历史拒绝轴是否已经关闭，并在 `historical_target_closure.axes` 中为每个轴输出 `closed` 和 `basis`。若无法确认所有历史轴都已关闭，不能输出 `pass=true` / `final_decision=deliver`。
如果需要 revise，必须输出 `blocking_stage`、`stage_assessment`、`direction` 和 `parameter_suggestions`，本地会把它们转成候选；如果建议与本地硬指标冲突，本地会记录 conflict。
`stage_assessment` 必须先判断本地 `blocking_stage` 是否真实存在；如果没有当前阻塞阶段，必须在 `basis` 中说明你已独立检查图像和关键视觉指标；`suggestion_target_stage` 必须等于当前 `blocking_stage`，没有阻塞阶段时必须为 null。
